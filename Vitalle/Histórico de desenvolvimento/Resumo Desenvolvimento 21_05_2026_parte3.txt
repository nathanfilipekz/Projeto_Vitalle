1. CORREÇÃO RLS — TABELA tenants
============================================================
Problema:
  Ao tentar cadastrar o primeiro usuário via tela de
  registro, o Supabase retornava:
    "new row violates row-level security policy for table
     'tenants'"

Causa raiz:
  A tabela public.tenants tem RLS habilitado. O frontend
  usa a anon key do Supabase para fazer INSERT direto, mas
  as policies mvp_anon (criadas na migration 008 de 20/05)
  estavam ausentes ou incompletas no banco.

Solução (SQL aplicado no Supabase SQL Editor):
  -- Remove antigas para evitar conflito
  DROP POLICY IF EXISTS tenants_mvp_anon      ON public.tenants;
  DROP POLICY IF EXISTS users_mvp_anon        ON public.users;
  DROP POLICY IF EXISTS doctors_mvp_anon      ON public.doctors;
  DROP POLICY IF EXISTS patients_mvp_anon     ON public.patients;
  DROP POLICY IF EXISTS appointments_mvp_anon ON public.appointments;

  -- Recria com FOR ALL (SELECT + INSERT + UPDATE + DELETE)
  CREATE POLICY tenants_mvp_anon ON public.tenants
    FOR ALL TO anon, authenticated
    USING (true) WITH CHECK (true);

  CREATE POLICY users_mvp_anon ON public.users
    FOR ALL TO anon, authenticated
    USING (true) WITH CHECK (true);

  CREATE POLICY doctors_mvp_anon ON public.doctors
    FOR ALL TO anon, authenticated
    USING (true) WITH CHECK (true);

  CREATE POLICY patients_mvp_anon ON public.patients
    FOR ALL TO anon, authenticated
    USING (true) WITH CHECK (true);

  CREATE POLICY appointments_mvp_anon ON public.appointments
    FOR ALL TO anon, authenticated
    USING (true) WITH CHECK (true);

Observação:
  Essas policies são temporárias (MVP). Quando o backend
  NestJS estiver autenticando via JWT, elas devem ser
  dropadas (ver pendência A.5 dos resumos anteriores).

2. ABA PACIENTES — CONEXÃO COM SUPABASE
============================================================
A página de Pacientes estava 100% mockada (array local
com 6 pacientes fictícios). Foi completamente reescrita
para usar dados reais do banco Supabase.

Mudanças na UX:
  - REMOVIDO: botão "Filtros" (não existia funcionalidade
    real por trás dele).
  - MANTIDO e melhorado: campo de busca — pesquisa por
    Nome (parcial, case-insensitive), CPF (dígitos) ou
    Telefone (dígitos) via OR no Supabase.
  - Busca com debounce de 300 ms para não disparar
    requisição a cada tecla.
  - Botão × para limpar a busca rapidamente.
  - Mensagem de contagem de resultados ao buscar.
  - "Novo Paciente" abre o NewPatientModal já existente;
    ao cadastrar, lista e stats são atualizados sem reload.

Estatísticas (reais, sem mocks):
  - Total de Pacientes: count de is_active=true.
  - Novos este mês: count com created_at >= início do mês.
  - Card "Aniversariantes" removido (dados insuficientes
    no MVP; maioria dos pacientes sem date_of_birth).

Cards de paciente:
  - Dados reais: nome, CPF formatado, telefone formatado,
    e-mail (se existir), data de nascimento (se existir),
    badge Ativo/Inativo.
  - Botões "Ver prontuário" e "Ver detalhes" mantidos
    (sem ação ainda — placeholder para fase futura).
  - Empty state amigável com link direto para cadastro.
