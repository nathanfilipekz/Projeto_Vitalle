2. medical-records-service.ts
============================================================
Arquivo: frontend/src/services/medical-records-service.ts

Interfaces:
  MedicalRecordPatient   { id, name, cpf, phone, date_of_birth }
  MedicalRecordRow       (todos os campos + patients: join)
  CreateMedicalRecordPayload / UpdateMedicalRecordPayload
  AppointmentHistoryRow  { id, date_time, status, type, notes }

Funções:
  resolveDoctorId(tenantId, userId)
    → SELECT id FROM doctors WHERE user_id=? AND tenant_id=?
    → Necessário pois medical_records.doctor_id é NOT NULL e
      o auth store armazena user.id (users table), não doctor.id

  listMedicalRecords(tenantId)
    → SELECT *, patients(id,name,cpf,phone,date_of_birth)
    → ORDER BY created_at DESC, is deleted_at null

  getMedicalRecordsByPatient(tenantId, patientId)
    → Igual mas filtrado por patient_id

  searchMedicalRecords(records[], query)  [client-side]
    → Filtra por name.toLowerCase, cpf digits, phone digits

  createMedicalRecord(payload)    → INSERT + retorna com join
  updateMedicalRecord(id, ...)    → UPDATE + retorna com join
  deleteMedicalRecord(id, ...)    → soft-delete (deleted_at)

  getPatientAppointmentHistory(tenantId, patientId, limit=20)
    → SELECT id,date_time,status,type,notes FROM appointments
    → ORDER BY date_time DESC
