2. TAREFA 9 — npm install + prisma generate
============================================================
Situação encontrada:
  - node_modules já existia (instalação prévia completa).
  - O pacote "prisma" (CLI) estava com a pasta
    node_modules/prisma/build/ vazia — sem package.json,
    sem binário executável.
  - npx prisma generate falhou porque o npx tentou baixar
    prisma@7.8.0 (latest) e o sandbox não tem acesso à
    URL de checksum em binaries.prisma.sh (403 Forbidden).

Conclusão:
  node_modules está funcional para o runtime (@prisma/client
  5.22.0 presente em @prisma/*). O prisma generate precisa
  ser executado LOCALMENTE na máquina do desenvolvedor
  (ver seção 6), onde a rede está disponível e os engines
  já podem ter sido cacheados em ~/.cache/prisma.

  Comando local:
    cd Vitalle/backend
    npm install          # garante prisma CLI v5.12.1 local
    npx prisma generate  # gera o Prisma Client em node_modules/.prisma/client

6. COMO REGISTRAR A BASELINE LOCALMENTE
============================================================
Execute estes comandos uma única vez no seu terminal:

  cd Vitalle/backend

  # 1. Instalar dependências (inclui CLI do Prisma v5.12.1)
  npm install

  # 2. Gerar o Prisma Client (tipos TypeScript + query engine)
  npx prisma generate

  # 3. Registrar a baseline como "já aplicada" no Supabase
  #    (não re-executa o SQL — só marca na _prisma_migrations)
  npx prisma migrate resolve --applied "0_init"

  # 4. Confirmar que está tudo ok
  npx prisma migrate status

Resultado esperado de migrate status:
  "1 migration found in prisma/migrations
   0_init ... Already applied"

A partir deste ponto, novas migrations podem ser criadas
normalmente com:
  npx prisma migrate dev --name <nome_da_feature>
